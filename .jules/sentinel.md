## 2024-06-21 - [Prompt-Based Security Mitigations for Path Traversal and XSS]
**Vulnerability:** Path Traversal via the `[slug]` component in Markdown file generation (`create_file`) and Cross-Site Scripting (XSS) via unescaped user input rendered in HTML cards.
**Learning:** In purely prompt-based AI applications (skills built via markdown instructions), traditional code-level sanitization is not possible. The LLM handles the logic entirely. If an attacker inputs `../../../` into a project name and the LLM naively uses it in `create_file` path, a path traversal vulnerability occurs. Similarly, an attacker can input `<script>alert(1)</script>` which may be rendered into an HTML response card.
**Prevention:** Security mitigations must be codified as explicit, mandatory sanitization instructions for the LLM within the skill definition Markdown files. We instruct the LLM to strictly sanitize filenames (e.g. keeping only alphanumeric characters and hyphens) and to always HTML-escape user data before rendering HTML cards.
