## 2024-03-24 - [Explicit Security Instructions in Prompt-Based Apps]
**Vulnerability:** XSS and Path Traversal risks in LLM-generated outputs (HTML and file paths).
**Learning:** In purely prompt-based applications (like this markdown skill), security boundaries cannot be enforced by traditional code logic. If the LLM generates HTML or file paths dynamically based on user input, it may unknowingly execute XSS payloads or write files outside intended directories if not explicitly instructed to sanitize.
**Prevention:** Security mitigations must be codified as explicit, mandatory instructions in the system prompt / skill definition. Always instruct the LLM to sanitize user inputs (e.g., escape HTML characters) and validate/sanitize file paths/slugs (remove slashes, dots, etc.) before generating artifacts.
## 2024-05-18 - [Preventing Stack Trace Leaks in Orchestrator Scripts]
**Vulnerability:** Information Disclosure via unhandled exceptions. External API calls (such as to the Anthropic SDK) can throw exceptions that, if uncaught, leak internal execution paths, configuration details, and stack traces to logs or directly to users.
**Learning:** In automation orchestrators built using Python or Node.js, failing securely is a mandatory defense-in-depth practice. Relying solely on the "happy path" for external network calls leaves the application vulnerable to informative crashes.
**Prevention:** Always wrap external API calls (e.g., `client.messages.create`) in `try...except` (Python) or `try...catch` (Node.js) blocks. Ensure that any logged or returned error messages are generic (e.g., "An internal communication error occurred") and never expose the raw error object or stack trace.
