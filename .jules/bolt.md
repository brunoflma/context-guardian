## 2026-06-22 - Replacing O(N) token heuristic with O(1) exact usage tracking
**Learning:** In the `automation-orchestrator.md` code embedded within the Context Guardian skill, the original token tracking approach used a heuristic function `estimate_tokens` to iterate over the entire message history on every single turn to estimate token usage in O(N) time. However, this is computationally inefficient as the session grows and inherently inaccurate. Anthropic's API responses inherently return the exact exact `input_tokens` and `output_tokens` usages.
**Action:** Always prefer caching and accumulating exact tracking metrics returned by API usages (e.g. `response.usage`) directly into state variables, checking these limits in O(1) time rather than re-estimating them over entire histories iteratively.
